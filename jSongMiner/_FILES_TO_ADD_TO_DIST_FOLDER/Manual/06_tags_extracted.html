EXPLANATION OF THIS SECTION

This section lists all of the metadata field labels currently extracted by jSongMiner. They are divided based on whether each field is output to a Song, Artist or Album resource file. These may or may not be packaged together into a single file, based on user configuration settings. The field labels are also divided within each of these three resource types based on the source from which a given field's value is extracted from. These sources are listed here exactly as they are named by jSongMiner.

Note that there is some redundancy between field labels, across both resource file types and sources. This is intentional, as in the latter case it serves as insurance against erroneous data returned by a single source, and in the former case it enables links to be made between the different types of files.

All or some of these fields may be present for a given extraction, depending on user settings and on what is available for the given resource. The resource types, sources and field labels are listed in the order that they are extracted and reported by jSongMiner. The field labels are specified exactly here as they are named by jSongMiner, with one exception: the "..." notation that is used in this document after some field labels indicates that additional text may be appended to the field label by jSongMiner depending on the particular resource being processed and/or extraction circumstances.

The metadata labels stored in the actual text and XML files output by jSongMiner consist of the source and the field label concatenated together. More precisely, they consist of the source (as it appears below), followed by ": ", followed by the field label (as it appears below). So, for example, one metadata label saved by jSongMiner is: "Echo Nest API: Song Title". Note that this will also likely be URL encoded (UTF-8), depending on user settings.

Note that all of this metadata may also be converted into and saved as qualified and/or unqualified Dublin Core-labelled fields by jSongMiner, based on user configuration settings. These Dublin Core-formatted fields incorporate all of the fields below, and do not incorporate any additional data values. See the section of the manual on the Dublin Core mappings used by jSongMiner for details of the mappings between the fields labels listed below and the corresponding Dublin Core field labels, and for general information on how jSongMiner presents Dublin Core-formatted fields.
