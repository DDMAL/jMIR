Tutorial

This section of the manual provides a brief applied introduction on using the jMusicMetaManager software.

  1. After installing the jMusicMetaManager software (see the Installation section of the manual), run it by double clicking on the jMusicMetaManager.jar file or by running it from a command prompt (see the Using the Software section).
  2. The software will start with the Options Panel displayed. This is where the user may choose the source of the metadata to analyze, the location to save the generated reports, the parameters to use in error detection analysis and which reports to generate. This tutorial will assume the use of the default settings, although the user may wish to deviate from these settings when actually applying the jMusicMetaManager to real situations (see the Hints and Suggestions section).
  3. Note the default paths at the top of the Options Panel. A sample Apple iTunes file that is provided with jMusicMetaManager is the default source of metadata to analyze, and the generated report root will be saved as WorkingDirectory/anaylysis_output_01.html by default. This iTunes file can be imported into the iTunes software itself if the user wishes to examine its contents. Note that no MP3 files will be directly analyzed under the default settings, as no such files are provided with the jMusicMetaManager installation, and all needed metadata is provided by the default iTunes file.
  4. Analysis of the iTunes file can be initiated by pressing the BEGIN METADATA ANALYSIS button at the bottom of the Options Panel. A progress bar will be briefly displayed while the file is analyzed using the default settings.
  5. The focus of jMusicMetaManager's GUI will automatically switch to the Report Panel, which will display the results of the processing of the default iTunes file. The results will also be saved as a set of HTML files to the location specified in the Options Panel, the root of which is WorkingDirectory\anaylysis_output_01.html by default. These results can either be viewed directly in the Report Panel or viewed using any web browser.
  6. The names of the generated reports are listed on the left of the Report Panel, and the currently selected report is on the right. Clicking on the name of a report on the left will cause it to be displayed on the right. These reports each either provide various profiles of the music collection or highlight probable metadata errors contained within it.
  7. The user should review these reports to get a feel for them and the kind of information that they provide. To give just a very few examples of the information provided in the reports:
    • The software correctly notes that Charles Mingus is present as both "Mingus, Charles" and "Charles Mingus".
    • The software correctly notes that the Borodin String Quartet is present as both "The Borodin String Quartet" and "Borodin String Quartet".
    • The software correctly notes that the track "Les Bourgeois" by Jacques Brel is present twice, and its title is mispelled as "Les Borgeois" once.
    • Note that "I Am A Man Of Constant Sorrow" occurs twice, "I Am A Man Of Constant Sorrow (Instrumental)" occurs twice and "String Quartet No. 8 In C minor Op. 110" appears thrice, but jMusicMetaManager correctly avoids detecting them as a redundant copies, since they are different versions of the pieces.
    • Note that several possible errors are listed in the "Probable errors in title field report" that are not in fact errors. This is an example of how false positives can sometimes be reported. They are written here to show that these titles are similar, and to give the user the opportunity to verify for him or herself whether the titles should in fact be the same.
    • etc.
