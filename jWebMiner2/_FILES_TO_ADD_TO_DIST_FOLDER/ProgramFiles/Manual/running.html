Using the Software

Running the Software

A file named "jWebMiner.jar" is produced upon installation. The simplest way to start jWebMiner is to simply double click on this file.

The software may also be accessed via the command line (e.g. the DOS prompt). To access DOS under Windows, for example, go to the Start Menu, select Run and type "cmd". Then use the "cd" command to move to the directory that contains the jWebMiner.jar file. In that directory, type:

java -jar jWebMiner.jar

It should be noted that the Java Runtime Environment does not always allocate sufficient memory for jWebMiner to process large music collections. Running jWebMiner using either of the above two methods could therefore result in an out of memory error (although this is relatively rare).

It is therefore sometimes preferable to manually allocate a greater amount of memory to jWebMiner before running it. 250 MB should be more than enough for most situations. This can be done by entering the following at the command prompt:

java -ms16M -mx250M -jar jWebMiner.jar

Note that your computer must be connected to the internet for jWebMiner to work. It may also be necessary to modify your firewall to allow it access.

Using the Software's Interface

The jWebMiner interface is divided into several panels that may be accessed via tabs below the menu bar. Each of these panels is described in a separate section in this manual, as described below. The menu bar itself allows access to help functionality as well as to the Network Search Dialog Box, which allows users to perform side by side searches using multiple web services and see the details of the returned hits. The EXTRACT FEATURES button appears below all of the panels, and it initiates feature extraction using the setting in each of the panels.

The Search Words Panel allows the user to choose the the text phrases (strings) that feature extraction is to be based on, to define synonyms amongst these strings and to decide whether the extraction is to involve co-occurrence or cross-tabulation analysis. The Required Filter Words Panel allows users to specify strings that must be present on a page in order for it to be counted as a hit and the Excluded Filter Words Panel allows them to specify words that may not be present if it is to be counted as a hit. The Site Weightings Panel allows users to specify specific sites that will be searched (including the possibility of the entire available network) and to set variable weighting schemes that will give particular sites greater influence than others on the final feature values. The Options Panel allows users to select amongst a variety of preferences. Finally, the Results Panel shows the results of feature extractions, which may include a wide variety of information, depending on the settings in the Options Panel. The Results Panel may also be used to save the results in a variety of file formats.

A tutorial is available to give a brief applied introduction to running the software.

Error Messages and Other Problems

jWebMiner generates informative error messages when any problems occur. The reported problems can most often be resolved by following the instructions in the error message. If a problem is encountered that cannot be resolved in this manner, then the Problems and Solutions section of the manual should be consulted.

Note that each query will be submitted to a given web services up to 3 times if it is unsuccessful before it is reported to the user, as sometimes the service is simply busy at the moment that a particular query is submitted. If a query fails three times consecutively then the user is presented with the option of either trying up to another 3 times or cancelling the feature extraction.

The results returned by web services are sensitive to the particularities of the search settings and query strings used. The Hints and Suggestions section of the manual can be consulted for tips on improving results.
