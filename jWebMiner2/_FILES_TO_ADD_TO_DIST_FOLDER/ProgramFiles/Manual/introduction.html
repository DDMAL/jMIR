Introduction

jWebMiner is an open source software system for automatically extracting cultural features from the web for use in classifying and/or measuring similarity between text terms. It is designed with the particular needs of music research in mind, but may easily be used in many other areas of research.

At its most basic level, the software operates by using web services to extract hit counts from search engines. Functionality is available for calculating a variety of statistical features based on these counts, for variably weighting web sites or limiting searches only to particular sites, for excluding hits that do or do not contain particular filter terms, for defining synonym relationships between search strings, and for applying a number of additional search configurations. A particular emphasis has also been placed on extensibility so that new functionality can be easily added.

jWebMiner can parse terms from newline delimited text files, from class names found in Weka ARFF or ACE XML files, or from a variety of different fields of Apple iTunes XML files. They can also be entered manually by the user into the jWebMiner GUI.

Detailed results and configuration records can be saved in HTML reports. Final feature values can also be exported to ACE XML, Weka ARFF or newline delimited text files.

jWebMiner was developed as part of the jMIR music classification and similarity analysis software suite, and may be used either as part of this suite or independently.

An academic summary of jWebMiner is available in this paper from the 2007 International Conference on Music Information Retrieval. Note that this publication may be viewed using any web browser, but not directly from jWebMiner's on-line help.

Please contact Cory McKay at cory.mckay@mail.mcgill.ca with any bug reports or questions relating to either jWebMiner or jMIR in general.

Summary of the Software's Functionality

The following list highlights some of the key functionality of the jWebMiner software:

  • Ability to automatically formulate and perform large numbers of queries.
  • Ability to automatically process hit counts into feature values using one of six different scoring functions and four different types of normalization.
  • Ability to save feature values as ACE XML, Weka ARFF or newline delimited text files to facilitate post-processing.
    • Feature values and other reports can also be saved as HTML files to facilitate human readability.
  • Ability to perform both co-occurrence oriented queries (formulating queries using all possible pairs of search strings from one set of search strings) and cross tabulation oriented queries (formulating queries using each search string from one set of search strings paired with each possible search string from another set of search strings).
  • Ability to use multiple web services to perform queries and combine the results across web services.
  • Ability to define synonyms between different search strings so that they will be considered equivalent and their results will be combined.
  • Ability to specify required filter words that must be present in a web page in order for it to be counted as a hit.
    • Pattern-based required filter words may also be used. These are formulated using the primary and/or secondary search string specific to each particular query.
  • Ability to specify excluded filter words that may not be be present in a web page in order for it to be counted as a hit.
  • Ability to limit searches to specific sites as well as or instead of the whole available network.
    • Relative weightings taken into account during feature calculation may also be assigned to specific sites and/or the whole available network.
  • Ability to set a variety of additional parameters that influence search results.
    • These include whether queries are treated as literal, whether pairs of strings are searched using a boolean OR or a boolean AND, whether non-matching similar hits are counted, whether similar hits are suppressed, whether adult content is suppressed, language filters, country filters, file type filters and the ability to specify which web service(s) are used and which regional servers they use.
  • Both feature values and a variety of intermediate reports can be automatically generated in HTML format.
    • Intermediate reports can include tabulated hit counts for all queries submitted to all web services (including separate values for different web services and/or web sites), these hit counts after each stage of processing, the actual queries submitted to each web service and the details of the search parameters used.
  • A search dialog box that allows users to perform probe searches using various search configurations in order to view the actual pages returned as well as the hit counts. Results for different web services are shown side by side.
    • This, along with the intermediate reports, can be useful in debugging when experimenting with the best search parameterization for a particular task or when extending the software's functionality.
  • Ability to load and save settings for search strings, synonyms, filter words, site limitations and site weightings.
  • Easy-to-use GUI-based interface that makes it easy to set search parameters.
  • Complete user manual.
  • Extensible software architecture and well-documented code.

Overview of the Manual

This manual may be viewed either with jWebMiner's online help or via any web browsing application (e.g. Mozilla Firefox). The latter option is preferable, as the jWebMiner online help display is not a fully equipped browser, and may not display external sites linked to by this manual as well as a full-fledged browser.

This manual is divided into a number of sections, as can be seen on the contents bar on the left. The Installation section gives instructions on installing jWebMiner and the Using the Software section gives instructions on how to run it. The Tutorial section provides a brief tour of the software that users can follow in jWebMiner using provided sample files, and provides an illustration of how the different parts of the interface are used together.

The jWebMiner interface is divided into several panels that may be accessed via tabs below the menu bar. Each of these panels is described in a separate section in this manual, as described below. The menu bar itself allows access to help functionality as well as to the Network Search Dialog, which allows users to perform side by side searches using multiple web services and see the details of the returned hits. The EXTRACT FEATURES button appears below all of the panels, and it initiates feature extraction using the setting in each of the panels.

The Search Words Panel allows the user to choose the the text phrases (strings) that feature extraction is to be based on, to define synonyms amongst these strings and to decide whether the extraction is to involve co-occurrence or cross-tabulation analysis. The Required Filter Words Panel allows users to specify strings that must be present on a page in order for it to be counted as a hit and the Excluded Filter Words Panel allows them to specify words that may not be present if it is to be counted as a hit. The Site Weightings Panel allows users to specify specific sites that will be searched (including the possibility of the entire available network) and to set variable weighting schemes that will give particular sites greater influence than others on the final feature values. The Options Panel allows users to select amongst a variety of preferences. Finally, the Results Panel shows the results of feature extractions, which may include a wide variety of information, depending on the settings in the Options Panel. The Results Panel may also be used to save the results in a variety of file formats.

A section providing hints and suggestions is also available to give ideas on how to use jWebMiner in such a way that results will be optimized, and an additional section provides a description of known problems and solutions.

Finally, this manual includes a section describing the licenses under which this software can be used and distributed, a section describing version updates and a section providing contact information for help requests and bug reports. There is also a section providing guidelines on extending the software for those interested in expanding and/or customizing jWebMiner.
